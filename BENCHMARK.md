# 벤치마크

`swift run PriTypeBenchmark`으로 측정. Apple M4 Pro, macOS 26.4, Swift 6.2, Debug 빌드 기준.

## 한자 검색

| 항목 | 수치 |
|------|------|
| 사전 로딩 (Cold, 1회) | 1,251ms |
| 검색 20키 (Cold) | 0.42ms |
| 검색 20키 (Cached) | 0.01ms |
| 검색 버스트 10,000회 | 0.98μs/op |
| 캐시 미스 순환 (40키 × 100회) | 2.92ms |

사전은 첫 한자키 입력 시 lazy 로딩되며, 이후 메모리에 상주한다.

## 자모 특수문자 검색

| 항목 | 수치 |
|------|------|
| JSON 최초 로딩 | 0.49ms |
| 전체 14키 검색 | 0.02ms |
| 버스트 10,000회 (ㅁ) | 1.74μs/op |
| 초성 변환+검색 10,000회 (ᄆ→ㅁ) | 1.73μs/op |

초성 자모(U+1100~)에서 호환 자모(U+3131~)로의 변환 오버헤드는 측정 한계 이하.

14개 호환 자모와 14개 초성 자모의 검색 결과가 모두 일치함을 검증(390개 특수문자).

## 동시성

| 항목 | 수치 |
|------|------|
| 순차 5,000회 | 45.82ms, 에러 0건 |
| 동시 5,000회 × 8스레드 | 166.20ms, 에러 0건 |

캐시는 `NSLock`으로 보호된다. IMK는 메인 스레드에서만 호출하므로 실제 경합은 발생하지 않으나, 방어적으로 동기화를 적용했다.

## 커서 좌표 검증 (isValidCursorRect)

| 입력 | 판정 |
|------|------|
| (607, 637, w=1, h=19) — 정상 | valid |
| (458, 978, w=0, h=18) — AX fallback | valid |
| (0, 0, 0, 0) — Chromium 실패 | invalid |
| (1.6e-314, 95886, ...) — Chromium 쓰레기값 | invalid |
| (-100, 500, ...) — 음수 좌표 | invalid |
| (100000, 500, ...) — 비현실적 좌표 | invalid |

9개 케이스 전체 통과. 버스트 100,000회에서 0.24μs/op.
